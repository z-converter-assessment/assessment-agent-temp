# docs/temp/

임시·외부 공유 자료 모음 (repo 간 협의 input, 제안서, 회신 메모 등). 디렉토리 위치는 영구, 안의 파일은 임시 — 자유 작성·삭제.

## 문서 교환 메커니즘 (repo 간 inbox)

에이전트 repo 와 엔진 repo 가 각자 `docs/temp/` 를 소유하고, 이 디렉토리가 상대가 나에게 쓰는 inbox 역할을 한다.

- 보내기: 상대에게 전할 문서는 상대 repo 의 `docs/temp/` 에 파일로 놓는다(내 repo 가 아니라).
- 받기: 내 `docs/temp/` 에 들어온 문서는 읽고, 채택할 내용을 내 repo 영구 문서(계약 문서·설계 문서 등)로 흡수한 뒤 그 임시 파일을 삭제한다. inbox 에 원본을 쌓아두지 않는다.
- 답장: 회신도 같은 규칙 — 상대 repo 의 `docs/temp/` 에 새 파일로 놓는다.

즉 각 파일은 "받은 쪽이 읽고 흡수하면 사라지는" 일회성 메시지다. 지속 가치가 있는 결정·합의는 영구 문서로 격상하고, 임시 파일은 지운다.

## 작성 규약

- 본문 첫 줄에 자료 성격 명시 — 임시·외부 공유·삭제 자유.
- self-contained 필수: 상대 repo reader 가 내 repo 의 다른 파일을 볼 수 없으므로, 코드 경로(`src/...`)나 내 repo 영구 문서 참조를 인용하지 않는다. 필요한 정책·구조는 문서 안에 풀어 쓴다. 코드 위치는 "어느 모듈·어느 책임" 수준으로 추상화한다.
- 내 repo 영구 문서·코드가 이 디렉토리 안 파일을 역으로 인용하지 않는다(임시 파일에 대한 의존 0).
- `docs/README.md` 인덱스나 `.claude/CLAUDE.md` 문서 인덱스에 등록하지 않는다.

## 처리

- 채택: 결정·합의를 영구 위치(계약 문서·설계 문서 등)로 격상하고, 이 디렉토리 안 파일은 삭제.
- 폐기: 협의 결렬·자료 폐기는 그냥 삭제.
